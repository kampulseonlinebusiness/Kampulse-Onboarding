# Kampulse Recruitment & Employee Onboarding Portal

Build a complete, production-ready Recruitment & Employee Onboarding Web Application for Kampulse Handling Solutions Ltd.

## Core Requirement

The entire application must be self-contained.

Do NOT use Supabase, Firebase, MongoDB Atlas, PlanetScale, Neon, Railway, Appwrite, PocketBase, or any Backend-as-a-Service.

Use PostgreSQL as the application's database, but configure it as part of this project so the entire application is self-contained and deployable as one project. The application should not rely on any third-party cloud database service.

All uploaded files must be managed by the application itself using local persistent storage. The database should store only metadata and file paths.

The final project should contain:

- Frontend
- Backend API
- PostgreSQL database configuration
- Authentication
- Admin Dashboard
- File Upload System
- PDF Generator
- Email notification module

Everything should belong to one complete project.
